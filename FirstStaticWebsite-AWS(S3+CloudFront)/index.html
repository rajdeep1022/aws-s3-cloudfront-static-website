What is Cloud Computing?

Cloud computing provides on-demand compute, storage, and networking resources over the internet so you can build, deploy, and scale applications without managing physical servers.

For static websites the cloud offers managed object storage (S3) and global delivery (CloudFront) ‚Äî which makes hosting fast, secure, and inexpensive.

Benefits

  • Low maintenance
  • Scales automatically
  • Pay for what you use
  • Global delivery & caching

What is Amazon S3?

Amazon Simple Storage Service (S3) is object storage built to store and retrieve any amount of data from anywhere. For static sites S3 can host your HTML, CSS, JS and assets directly.

Static website hosting

Configure a bucket to serve website content with public read access (or via CloudFront for security).

Durability

Designed for 99.999999999% durability and versioning support to protect your files.

Lifecycle rules

Automatically move or expire objects to manage cost and compliance at scale.

What is Amazon CloudFront?

CloudFront is a Content Delivery Network (CDN) that caches content at edge locations around the world ‚Äî speeding up delivery and reducing latency for end users.

Why use CloudFront?

  • Fast global distribution
  • TLS & HTTPS support
  • Custom domains & edge caching rules
  • Integrates with S3, Lambda@Edge, and WAF

Steps to deploy a static website

UI-only checklist: follow each step in the AWS Console or CLI to deploy your site.

  1. Create an S3 bucket

    Choose a unique name and region. Enable static website hosting or keep private if using CloudFront.

  2. Upload your site files

    Add HTML, CSS, JS, and images. Set object metadata (content-type) correctly.

  3. Set permissions

    Use CloudFront in front of S3 for secure access or configure bucket policy for public reads (not recommended for production).

  4. Create CloudFront distribution

    Choose your S3 bucket as the origin, enable TLS, set cache behaviors and default root object.

  5. Point domain & configure HTTPS

    Use Route 53 or your DNS provider; use ACM (AWS Certificate Manager) for TLS with CloudFront.

  6. Test & invalidate cache

    Ensure content serves correctly from edges. Use invalidation when you need immediate updates.
